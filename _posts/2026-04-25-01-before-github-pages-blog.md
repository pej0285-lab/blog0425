# GitHub Pages 블로그를 만들기 전에 알아둘 것

블로그를 직접 운영해보고 싶다면 선택지는 많다. 티스토리, 브런치, Velog, Medium처럼 바로 글을 쓸 수 있는 서비스도 있고, WordPress처럼 서버와 관리 화면을 함께 쓰는 방식도 있다. 그런데 개발 공부를 하거나 기록을 오래 남기고 싶다면 GitHub Pages와 Jekyll 조합은 꽤 매력적인 선택이다.

이 시리즈에서는 GitHub에 새 저장소를 만들고, Jekyll 테마를 연결하고, GitHub Pages로 배포하는 과정을 차근차근 정리한다. 최종 목표는 내가 직접 관리하는 정적 블로그를 만드는 것이다.

## GitHub Pages란?

GitHub Pages는 GitHub 저장소에 있는 파일을 웹사이트로 공개해주는 기능이다. HTML, CSS, JavaScript 파일을 올려두면 GitHub가 정적 사이트로 호스팅해준다.

일반적으로 블로그를 운영하려면 서버를 빌리고, 웹 서버를 설정하고, 배포 환경을 관리해야 한다. 하지만 GitHub Pages를 쓰면 이런 운영 부담이 크게 줄어든다. 저장소에 파일을 올리고, Pages 설정을 켜면 웹사이트 주소가 생긴다.

예를 들어 GitHub 계정이 `username`이고 저장소 이름이 `my-blog`라면 보통 다음과 같은 주소로 블로그가 열린다.

```text
https://username.github.io/my-blog/
```

## Jekyll이 하는 일

GitHub Pages는 정적 사이트를 보여준다. 그런데 매번 HTML 파일을 직접 작성하는 것은 번거롭다. 여기서 Jekyll이 등장한다.

Jekyll은 Markdown으로 작성한 글과 설정 파일을 읽어서 HTML 사이트로 만들어주는 정적 사이트 생성기다. 우리는 글을 Markdown으로 쓰고, Jekyll은 그 글을 블로그 페이지로 변환한다.

Jekyll이 해주는 대표적인 일은 다음과 같다.

- Markdown 글을 HTML 페이지로 변환한다.
- `_config.yml` 설정을 바탕으로 사이트 정보를 관리한다.
- `_posts` 폴더의 글을 날짜순 블로그 글로 처리한다.
- 레이아웃과 테마를 적용해 일관된 디자인을 만든다.

## 테마를 쓰는 이유

Jekyll은 기본 기능만으로도 사이트를 만들 수 있지만, 디자인을 처음부터 직접 만들면 시간이 많이 든다. 그래서 이미 만들어진 테마를 사용한다.

이 블로그에서는 `Just the Docs` 테마를 사용할 것이다. 원래는 문서 사이트에 잘 어울리는 테마지만, 검색 기능과 사이드바 구조가 깔끔해서 기술 블로그나 학습 기록용 블로그에도 잘 맞는다.

Just the Docs의 장점은 다음과 같다.

- GitHub Pages와 함께 쓰기 좋다.
- 검색 기능이 기본으로 제공된다.
- Markdown 문서를 깔끔하게 보여준다.
- 사이드바 내비게이션을 구성하기 쉽다.
- 모바일에서도 읽기 편하다.

## 정적 블로그의 장단점

GitHub Pages와 Jekyll로 만드는 블로그는 정적 블로그다. 정적 블로그는 서버에서 매번 페이지를 생성하지 않고, 미리 만들어진 HTML 파일을 보여준다.

장점은 분명하다.

- 서버 관리 부담이 적다.
- 속도가 빠르다.
- Git으로 글과 변경 이력을 관리할 수 있다.
- 비용 없이 시작할 수 있다.
- Markdown 기반이라 글을 백업하고 옮기기 쉽다.

하지만 단점도 있다.

- 댓글, 로그인, 관리자 페이지 같은 동적 기능은 직접 붙여야 한다.
- Ruby, Bundler, Jekyll 같은 도구에 익숙해져야 한다.
- 처음 설정할 때는 일반 블로그 서비스보다 복잡하게 느껴질 수 있다.

## 앞으로 만들 결과물

이 시리즈가 끝나면 다음과 같은 구조의 블로그를 갖게 된다.

```text
my-blog/
├─ .github/workflows/pages.yml
├─ _config.yml
├─ Gemfile
├─ index.md
├─ blog/
├─ docs/
└─ _posts/
```

그리고 글은 `_posts` 폴더 안에 Markdown 파일로 작성한다.

```text
2026-04-25-my-first-post.md
```

GitHub에 커밋하고 푸시하면 GitHub Actions가 Jekyll 사이트를 빌드하고, GitHub Pages에 자동으로 배포한다.
